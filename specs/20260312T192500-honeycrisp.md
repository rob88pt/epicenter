# Honeycrisp — Simple Apple Notes Clone

**Date**: 2026-03-12
**Status**: Implemented
**Parent**: `specs/20260311T224500-apple-notes-archetype.md`
**Priority**: Build first (simpler, well-defined scope)

## Goal

Faithful Apple Notes clone. Three-column layout (sidebar, note list, editor). Folders, pinned notes, rich text editing. Nothing fancy—just a clean, working notes app.

## Why Build First

1. Scope is 100% defined—Apple Notes is the spec
2. Simpler schema (folders + notes, no experimental fields)
3. Proves the Sidebar + Tiptap + workspace pattern that Fuji rewrite will also need
4. Standalone value—a usable notes app in the Epicenter ecosystem

## Architecture

### Workspace ID: `epicenter.honeycrisp`

### Schema

```typescript
import {
  createWorkspace, defineKv, defineTable, defineWorkspace,
  type InferTableRow, DateTimeString, dateTimeStringNow, generateId,
} from '@epicenter/workspace';
import { indexeddbPersistence } from '@epicenter/workspace/extensions/sync/web';
import { type } from 'arktype';
import type { Brand } from 'wellcrafted/brand';

// ─── Branded IDs ──────────────────────────────────────────────────────────

export type NoteId = string & Brand<'NoteId'>;
export const NoteId = type('string').pipe((s): NoteId => s as NoteId);

export type FolderId = string & Brand<'FolderId'>;
export const FolderId = type('string').pipe((s): FolderId => s as FolderId);

// ─── Tables ───────────────────────────────────────────────────────────────

const foldersTable = defineTable(
  type({
    id: FolderId,
    name: 'string',
    'icon?': 'string | undefined',
    sortOrder: 'number',
    _v: 1,
  }),
);
export type Folder = InferTableRow<typeof foldersTable>;

const notesTable = defineTable(
  type({
    id: NoteId,
    'folderId?': FolderId.or('undefined'),
    title: 'string',
    preview: 'string',
    pinned: 'boolean',
    createdAt: DateTimeString,
    updatedAt: DateTimeString,
    _v: 1,
  }),
).withDocument('body', {
  guid: 'id',
  onUpdate: () => ({ updatedAt: dateTimeStringNow() }),
});
export type Note = InferTableRow<typeof notesTable>;

// ─── Workspace ────────────────────────────────────────────────────────────

export default createWorkspace(
  defineWorkspace({
    id: 'epicenter.honeycrisp',
    tables: { folders: foldersTable, notes: notesTable },
    kv: {
      selectedFolderId: defineKv(FolderId.or(type('null'))),
      selectedNoteId: defineKv(NoteId.or(type('null'))),
      sortBy: defineKv(type("'dateEdited' | 'dateCreated' | 'title'")),
      sidebarCollapsed: defineKv(type('boolean')),
    },
  }),
).withExtension('persistence', indexeddbPersistence);
```

- ❌ Checklists / hasChecklist detection
- ❌ Word count
- ❌ Focus mode
- ❌ Folder colors

- ❌ Editor font size preference

## Review

**Completed**: 2026-03-12

### Summary

Built a complete Apple Notes clone as a SvelteKit SPA in `apps/honeycrisp/`. Three-column layout with collapsible sidebar (folders), resizable note list (date-grouped with pinned section), and Tiptap rich-text editor backed by Y.Text via y-prosemirror. Workspace schema uses two tables (folders, notes) with branded IDs, optional folder assignment, and automatic `updatedAt` on document edits. Registered as an epicenter template.

### Deviations from Spec

- `_v` field uses number literal `1` in `.set()` calls rather than string `'1'` — the arktype schema resolves to number type at runtime despite string syntax in the definition.
- Move-between-folders not yet exposed in the UI — the workspace API supports it (`notes.update(id, { folderId })`) but no dropdown or drag-and-drop target exists yet. Noted in spec.
- Mobile Sheet drawer behavior comes free from `SidebarProvider` — no custom implementation needed.

### Follow-up Work

- Add move-to-folder UI (dropdown on note right-click or drag-and-drop)
- Folder drag reordering (sortOrder field exists but no drag UI)
- Sort notes by title/dateCreated (sortBy KV exists but no UI toggle)
