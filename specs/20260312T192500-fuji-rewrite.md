# Fuji Rewrite — Personal Content App

**Date**: 2026-03-12
**Status**: Implemented
**Parent**: `specs/20260311T224500-apple-notes-archetype.md`
**Priority**: Build second (after Honeycrisp proves the Sidebar pattern)

## Goal

Ground-up rewrite of Fuji as a personal content app with a 1:1 mapping to your blog. Entries are content pieces—articles, thoughts, ideas—organized by tags and type, displayed in a data table with an editor panel. Not a notes app. A personal CMS built on the workspace API.

## Current State (Fuji v1)

Fuji v1 exists at `apps/fuji/` with:
- Simple schema: `{ id, title, preview, pinned, createdAt, updatedAt }`
- Two-column layout using `Resizable.PaneGroup`
- Tiptap + y-prosemirror editor
- Timeline sidebar with date-grouped entries
- Known UI bugs: timestamp/button overlap, no Sidebar component

**Approach**: Rewrite in-place. Keep workspace ID `epicenter.fuji`. Schema bumps from `_v: 1` to `_v: 2`.

## Design Decisions (Locked)

| Decision | Choice | Rationale |
|----------|--------|-----------|
| **Table name** | `entries` | Neutral—covers articles, thoughts, ideas. Matches blog mental model. |
| **Folders** | No | Flat table. No hierarchy. Tags are the organizer. |
| **Tags** | `tags: string[]` | Array of strings. Multiple tags per entry. Used for filtering/grouping. |
| **Type** | `type: string[]` | Array of strings. Most entries have one type, but supports multiple. Categorization orthogonal to tags. |
| **Title** | Explicit, required | Always show a title input. No auto-titling. Blog posts have titles. |
| **Content** | Y.Text via `.withDocument('body')` | Rich text body, separate from table row. `preview` column stores first ~100 chars for table display. |
| **Primary view** | Data table (TanStack Table) | Flat table of all entries with columns, sorting, filtering. Similar to the Epicenter entries table view. |
| **Editor** | Side panel or expanded view | Clicking a row opens the editor. Tiptap + y-prosemirror on Y.Text. |
| **Timeline** | Available as a view mode | Default: data table. Can toggle to timeline view (date-grouped). |
| **Column order** | id, title, preview, type, tags, createdAt, updatedAt | id first, then content fields, then metadata at the end. |

## Schema

```typescript
import {
  createWorkspace, defineKv, defineTable, defineWorkspace,
  type InferTableRow, DateTimeString, dateTimeStringNow, generateId,
} from '@epicenter/workspace';
import { indexeddbPersistence } from '@epicenter/workspace/extensions/sync/web';
import { type } from 'arktype';
import type { Brand } from 'wellcrafted/brand';

// ─── Branded IDs ──────────────────────────────────────────────────────────

export type EntryId = string & Brand<'EntryId'>;
export const EntryId = type('string').pipe((s): EntryId => s as EntryId);

// ─── Tables ───────────────────────────────────────────────────────────────

const entriesTable = defineTable(
  type({
    id: EntryId,
    title: 'string',
    preview: 'string',
    'type?': 'string[] | undefined',
    'tags?': 'string[] | undefined',
    createdAt: DateTimeString,
    updatedAt: DateTimeString,
    _v: 2,
  }),
).withDocument('body', {
  guid: 'id',
  onUpdate: () => ({ updatedAt: dateTimeStringNow() }),
});
export type Entry = InferTableRow<typeof entriesTable>;

// ─── Workspace ────────────────────────────────────────────────────────────

export default createWorkspace(
  defineWorkspace({
    id: 'epicenter.fuji',
    tables: { entries: entriesTable },
    kv: {
      selectedEntryId: defineKv(EntryId.or(type('null'))),
      viewMode: defineKv(type("'table' | 'timeline'")),
      sidebarCollapsed: defineKv(type('boolean')),
    },
  }),
).withExtension('persistence', indexeddbPersistence);
```

### Schema Notes

- `type` (singular name, `string[]` value) — e.g. `["article"]`, `["thought", "personal"]`. Most entries have one type.
- `tags` (plural name, `string[]` value) — e.g. `["crdt", "open-source", "yjs"]`. Freeform tagging.
- `preview` — first ~100 chars of body content. Auto-derived when body changes (via `onContentChange` in editor).
- `pinned` is **removed** — v1 had it, v2 doesn't. Pinning is a notes-app concept. CMS uses sorting/filtering.
- `viewMode` KV — persists whether the user prefers table or timeline view.

### Migration (v1 → v2)

Existing Fuji v1 entries (`_v: 1`) need migration:
- Add `type: undefined` (new optional field)
- Add `tags: undefined` (new optional field)
- Remove `pinned` (or ignore — `_v: 2` schema won't read it)

## Review

### Summary of Changes

Ground-up rewrite of Fuji from a timeline notes app (v1) into a personal CMS (v2) with:

- **Schema v2**: Added `type` and `tags` (both `string[]`), removed `pinned`, explicit title. Each entry has a Y.Text body document via `.withDocument('body')`.
- **Sidebar** (`FujiSidebar.svelte`): Search input, type groups with counts, tag groups with counts, recent entries timeline. Uses `@epicenter/ui/sidebar` following Honeycrisp pattern.
- **Data Table** (`EntriesTable.svelte`): TanStack Table with sortable columns (title, preview, type, tags, createdAt, updatedAt), global text search, badge rendering for tags/type arrays. Row click opens editor.
- **Editor** (`EntryEditor.svelte`): Explicit title input, type/tag TagInput fields, Tiptap + y-prosemirror body, auto-derived preview (first 100 chars), back button, timestamp footer.
- **Timeline** (`EntryTimeline.svelte`): Date-grouped timeline view with empty state. Available as alternate view mode.
- **View mode toggle**: Table ↔ timeline, persisted in KV via `viewMode` key.
- **Keyboard shortcuts**: `⌘N` (new entry), `Escape` (back to table from editor), `⌘B` (sidebar toggle via SidebarProvider).
- **Template**: Updated `apps/epicenter/src/lib/templates/fuji.ts` to match v2 schema.

### Known Issues

- **YText vs YXmlFragment**: `EntryEditor.svelte` passes `Y.Text` to `ySyncPlugin` which expects `YXmlFragment`. This was inherited from v1's `editor.svelte` and works at runtime but produces a type error. Fixing requires changing the document type to `YXmlFragment` in the workspace schema.
- **Pre-existing type errors**: 2 errors in `packages/workspace` (`NumberKeysOf` not found, `updatedAt` index type), 2 in `packages/ui` (`Record` generic arity). Not from this rewrite.
